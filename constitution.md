# 项目宪法

本文件定义了本项目不可动摇的核心开发原则。所有 AI Agent 在进行技术规划和代码实现时，必须无条件遵循。

## 第一条：简单性原则（Simplicity First）

核心：遵循简洁高效的开发哲学，尽量减少不必要的抽象和依赖。

- 1.1（YAGNI）：只实现 `spec.md` 中明确要求的功能，避免任何不必要的功能添加。
- 1.2（框架优先）：在后端，必须优先使用 FastAPI 标准能力及其提供的功能进行开发；在前端，优先使用 Vue 3 和 Vite 的官方推荐工具和库。
- 1.3（反过度工程）：优先选择简单的函数和数据结构，避免过度复杂的架构和不必要的抽象层，尤其是在 API 设计和路由管理上。

## 第二条：测试先行铁律（Test-First Imperative）- 不可协商

核心：所有新功能或 Bug 修复，都必须从编写一个或多个失败的测试开始。

- 2.1（TDD 循环）：严格遵循 Red-Green-Refactor 循环。后端使用 `pytest`，前端使用 `vitest`。
- 2.2（表格驱动）：单元测试必须优先采用表格驱动测试风格。后端应覆盖不同请求和边界输入，前端应覆盖清晰且可验证的行为分支。
- 2.3（拒绝无意义 Mock）：优先编写集成测试，尽可能使用真实依赖和真实调用路径，避免仅依赖脱离上下文的 Mock 数据。

## 第三条：明确性原则（Clarity and Explicitness）

核心：代码的首要目的是让人类易于理解。

- 3.1（错误处理）：所有错误都必须被显式处理。FastAPI 后端使用 `HTTPException` 或明确的应用级错误响应；前端 Vue 应用使用 `try/catch` 等显式机制进行错误捕获和处理。
- 3.2（无隐式全局状态）：绝不允许使用全局变量传递业务状态。所有依赖都必须通过函数参数、明确模块边界，或通过 Pinia 等显式状态容器管理。

## 治理（Governance）

本宪法具有最高优先级，其效力高于任何 `AGENTS.md` 或单次会话中的临时指令。

- 所有开发者和团队成员必须遵循本宪法定义的原则。
- 若实现、模板、流程或局部规则与本宪法冲突，以本宪法为准。
- 对本宪法的修改必须是显式决策，不得在日常开发中被隐式弱化或绕过。
