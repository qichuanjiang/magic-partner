# 需求规格：sample-user-greeting

## 1. 背景

提供一个最小示例，演示规格驱动流程如何使用。

## 2. 目标

- 演示如何描述 API 行为。
- 演示如何定义验收标准。

## 3. 非目标

- 不涉及数据库变更。
- 不涉及认证/鉴权变更。

## 4. 用户场景

- 用户通过 `name` 请求问候语，并收到个性化返回。

## 5. 功能需求

- FR-1：`GET /api/greeting` 支持可选查询参数 `name`。
- FR-2：当 `name` 缺省时，默认使用 `World`。

## 6. API 契约

- 接口路径：`/api/greeting`
- 请求方法：`GET`
- 请求参数：`name?: string`
- 响应结构：`{ message: string, timestamp: number }`
- 错误码：查询参数格式非法时返回 `400`。

## 7. 数据模型 / 持久化

- 无。

## 8. 安全与权限

- 当前演示范围内为公开接口。

## 9. 风险与缓解

- 风险：响应结构不一致。
- 缓解：增加响应模型校验。

## 10. 发布与回滚

- 发布：部署后端服务。
- 回滚：回滚到上一版本镜像或标签。
